Forms

Form elements are styled accoring to their DOM Object type. It's not neccessary to add classes on them.


Text Inputs

<label for='mytext'>Input Text:</label> <input type='text' id='mytext'>

Disabled Inputs

<label class='disabled' for='mytextdisabled'>Input Text:</label> <input id='mytextdisabled' class='disabled' type='text'>
Note: Any kind of element can be disabled by adding the class .disabled

Validation Errors

Add the class error to inputs to mark a validation error.

Example: <label for='mytext'>Input Text:</label> <input class='error' type='text' id='mytext'>

Deletable Inputs

Add the class deletable to the input you wish to make deletable.
Add the "cancel" icon after the input. It will position automatically inside the input element as long as you have added the class "deletable" to the input.

Example: <label for='mytext'>Input Text:</label> <input class='deletable' type='text' id='mytext' value='Hello'><i class="material-icons">cancel</i> cancel

Search Box and similar action icons after inputs

Example <label for='mytext'>Input Text:</label> <input type='text' id='mytext' value='Hello'> <i class="material-icons md-24 action bg-green">search</i>

Add the class action to the icon element. Use a background color of your choice with the bg-[color] class.

Extenders and accordions

An extender consists of following elements:
.extender - is the container of the extender
.extender-header - is the clickable element and title.
.extender-content - holds the content, which can be collapsed or extended

Note: Extenders are collapsed by default. In order to load them opened, add the class extended to the .extender-header element.
Example1: Normal extender <div class='extender'> <h3 class='extender-header extended'> <i class='material-icons'>keyboard_arrow_up</i> Extend me </h3> <div class='extender-content'> Add some content here! </div> </div>

hiding elements on mobile devices

Following classes can be added to an element to hide content on mobile:

.hide-on-tablet-landscapeHides element on tablets (landscape and portrait) and smartphones
.hide-on-tablet-portraitHides element on tablets (portrait) and smartphones
.hide-on-mobileHides element on smartphones

Example 1: text is hidden on tablets(all orientations) and smartphone <p class='red hide-on-tablet-landscape'> <i class="material-icons">tablet</i> I will disappear on tablet-landscape </p>

Example 2: text is hidden on tablets(portrait orientation) and smartphone <p class='green hide-on-tablet-portrait'> <i class="material-icons">tablet_mac</i> I will disappear on tablet-portrait </p> Example 3: text is hidden on smartphone
